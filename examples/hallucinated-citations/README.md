# Worked Example: Hallucinated Citations In Professional Reports

This folder is a complete, inspectable run of the workflow on one public research question:

> How should a short briefing describe documented cases of AI-hallucinated citations in professional work without overstating the evidence?

It uses public sources only. Nothing here is confidential, private or unpublished.

Read it in this order:

1. [source-log.md](source-log.md): what was collected, and what each source can and cannot support.
2. [claim-ledger.md](claim-ledger.md): the candidate statements, each marked keep, revise or drop.
3. [review-gate.md](review-gate.md): what the skeptical reviewer and non-expert reader found.
4. [decision-log.md](decision-log.md): the human calls.
5. [draft.md](draft.md): the short prose built only from surviving claims.

Two things are worth noticing:

- **Claim C-002 was dropped.** A precise statistic appeared with the wrong source attached. It may be traceable elsewhere, but not in the source cited by the first draft.
- **Claim C-007 was narrowed.** It appeared during drafting as a broad frequency claim. The sources support documented cases, not a measured base rate.

If the workflow is worth anything, those two lines are why.
